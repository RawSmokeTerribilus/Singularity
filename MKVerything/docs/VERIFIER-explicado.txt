Aquí tienes el análisis del script verifier.py explicado de forma sencilla.
📅 **Actualización V2.beta:** 20 de febrero de 2026

🧐 ¿Qué es este archivo?

Este script es el **Auditor Forense** del proyecto. Su único trabajo es mirar archivos de vídeo con lupa molecular y decir si son "Válidos" o "Sospechosos". No modifica, no borra y no mueve nada. **Solo juzga**. Es el Juez Supremo que el rescatador llama para validar su trabajo.

**Filosofía Central:** "Ante la duda, el archivo NUEVO es culpable y el ORIGINAL no se toca".

---

🛠️ ¿Qué herramientas usa?

El verifier utiliza 3 poderes combinados:

1. **FFprobe** (escáner de rayos X): Lee metadatos sin reproducir el archivo completo
2. **mkvmerge -J** (lector de estructura MKV): Verifica la integridad estructural de contenedores MKV
3. **ffmpeg -xerror** (decodificador brutal): Prueba REAL de decodificación bit-a-bit (la verdadera prueba de fuego)

---

🧠 Sus Funciones (Habilidades Actuales)

Aquí te explico qué hace cada función del código:

### 1. El Notario (_log)

**Qué hace:** Registra todo lo que sucede.

**Detalles:**
- Escribe en `logs/security_audit.log` con timestamp
- Niveles de severidad: INFO, FAIL, WARNING, SUCCESS
- Cada línea queda documentada para auditoría futura

---

### 2. Los Escáneres Internos

#### _run_ffprobe(filepath)
Obtiene un JSON estructurado con TODA la información técnica del archivo:
- Códecs de vídeo y audio
- Duración exacta en segundos
- Resolución (ancho x alto)
- Idiomas de audio
- Información de subtítulos
- Bitrates

Si FFprobe falla (archivo muy corrupto), devuelve None.

#### _run_mediainfo(filepath)
Alternativa a FFprobe. Algunos archivos muy rotos que ffprobe no puede leer, MediaInfo SÍ puede.
Devuelve JSON con estructura similar a ffprobe.

#### _get_mkv_title_tag(filepath)
Lee el título interno grabado en los metadatos del MKV.
Ejemplo: Si el archivo se llama "Matrix.mkv", el título interno podría ser "The Matrix (1999)" o "Matrix RipByRarbg".

Esto es importante para detectar si el archivo está contaminado con spam.

---

### 3. El Juez Supremo (check_health)

**ES LA FUNCIÓN MÁS IMPORTANTE.**

Ejecuta 3 pruebas secuenciales. Si cualquiera falla, el archivo es SOSPECHOSO:

#### Prueba 1: Estructura MKV (mkvmerge -J)
Solo para archivos .mkv. Valida que el contenedor MKV sea parseable.
Si falla: Ya sabemos que está roto a nivel de estructura.

#### Prueba 2: Metadatos Legibles (ffprobe)
Valida que ffprobe pueda leer:
- Duración > 0 segundos
- Al menos 1 pista de vídeo
Si falla: El archivo es ilegible.

#### Prueba 3: Decodificación Completa (ffmpeg -xerror)
**La Prueba de Fuego Real.** Decodifica el archivo COMPLETO sin mostrar salida.
- Flag `-xerror`: Detiene al PRIMER error de decodificación encontrado
- Flag `-f null -`: No genera archivo de salida
- Si falla: El archivo tiene corrupción de datos que la decodificación detecta

**Resultado Final:** Solo devuelve True si pasa las 3 pruebas. Es casi imposible falsificar.

---

### 4. El Verificador de Extracción (check_iso_extraction)

**Caso de Uso:** Cuando extract.py extrae una ISO a MKV, llama a esta función.

**Qué hace:**
1. Verifica que el archivo MKV exista
2. Le pasa la prueba de salud completa (check_health)
3. Devuelve un veredicto en formato JSON

**Estructura del Resultado:**
```python
{
  "valid": True/False,
  "errors": ["CORRUPT_STRUCTURE", "FILE_NOT_FOUND"],
  "level": "SUCCESS" or "FAIL"
}
```

**Regla Crítica:** Si valid=False, extract.py NO borra la ISO original.

---

### 5. El Comparador (check_rescue)

**Caso de Uso:** Cuando universal_rescuer.py convierte un archivo, llama aquí.

**Input:** Archivo original + Archivo nuevo rescatado

**Proceso de Validación:**

1. **Existe el nuevo:** El archivo nuevo debe tener > 1KB
   
2. **Pistas de Audio:** Compara cantidad de audios
   - Si el original tenía 3 audios y el nuevo tiene 2 → ERROR
   - No se pierden pistas

3. **Pistas de Subtítulos:** Compara cantidad de subs
   - Si el original tenía subs y el nuevo no → ERROR

4. **Pistas de Vídeo:** Compara cantidad de streams de vídeo
   - Si el original tenía 1 y el nuevo tiene 0 → ERROR

5. **Resolución:** Compara ancho en píxeles
   - Si el nuevo es < 98% del ancho original → ERROR
   - Previene que un 1080p se convierta en 720p por error

6. **Duración:** Compara tiempo total
   - Tolerancia: ±5%
   - EXCEPCIÓN INTELIGENTE: Si el original duraba 0 segundos (roto) y el nuevo dura > 5 minutos → APROBADO (hemos reparado el archivo)

**Casos Reales:**
```
Original: 45:32 (2730 segundos)
Nuevo:    45:35 (2735 segundos)
Diferencia: 5 segundos within 5% tolerance → ✅ VÁLIDO

Original: 0:00 (corrupto, duraba 0)
Nuevo:    1:23:45
Resultado: ✅ VÁLIDO (hemos rescatado un archivo completamente roto)
```

**Estructura del Resultado:**
```python
{
  "valid": True/False,
  "level": "SUCCESS/FAIL",
  "errors": ["DUR_DIFF (45m vs 23m)", "RESOLUTION_DROP"],
  "details": {
    "note": "RECOVERED_FROM_CORRUPT"
  }
}
```

---

### 6. El Auditor de Metadatos (audit_file_metadata)

**Qué hace:** Detecta "basura" en los metadatos sin depender de APIs.

**Check 1: SPAM KEYWORDS**
Busca palabras prohibidas en el título interno:
```
"rarbg", "yify", "ettv", "www.", ".com", "torrent", 
"axxo", "brrip", "web-dl", "bluray", "rip", "x264", "x265",
"ac3-evo", "evo", "bypixel", "spam", "wolfmax"
```

Si encuentra una: Marca como `spam_detected: True`

**Check 2: MISMATCH DE TÍTULOS**
Compara nombre del archivo vs título interno usando similitud textual:
```
Archivo: Matrix.mkv
Título interno: The Matrix 1999
Similitud: Alta ✅

Archivo: Matrix.mkv
Título interno: Shrek (2001)
Similitud: Baja → title_mismatch: True ❌
```

**Resultado:**
```python
{
  "clean": True/False,
  "spam_detected": True/False,
  "title_mismatch": True/False,
  "spam_word": "rarbg"  # (si encontró algo)
}
```

### 7. Helpers Internos

#### _count_tracks_ffprobe(meta)
Cuenta cuántas pistas de cada tipo hay en los metadatos.
Devuelve: `{'video': 1, 'audio': 3, 'subtitle': 2}`

#### _get_width(meta)
Extrae la resolución de ancho (píxeles) del stream de vídeo.
Ejemplo: 1920 (para 1080p)

📋 **Flujo Real en el Proyecto**

1. **extract.py extrae ISO → MKV**
   ```
   result = verifier.check_iso_extraction(iso_path, mkv_path)
   if result["valid"] == True:
       # ✅ MKV está OK, pero ISO NUNCA se borra
       # La ISO se preserva siempre (son sagradas)
   else:
       print(f"❌ Extracción falló: {result['errors']}")
       # ISO se preserva, usuario revisa manualmente
   ```
   **Regla Inamovible:** Las ISOs NUNCA se borran. Son los "masters" del proyecto.

2. **universal_rescuer.py convierte Legacy (AVI/MP4/etc) → MKV**
   ```
   result = verifier.check_rescue(original_legacy, new_mkv)
   if result["valid"] == True:
       os.remove(original_legacy)  # ✅ OK, borrar el AVI/MP4/WMV
       # Se recupera espacio inmediatamente
   else:
       print(f"❌ Conversión falló: {result['errors']}")
       # Archivo legacy se preserva, usuario revisa manualmente
   ```
   **Regla Inamovible:** Los archivos legacy (AVI, MP4, WMV, etc.) SÍ se borran tras éxito verificado.
   **Seguridad:** Si check_rescue devuelve valid=False, el original se preserva intacto.

3. **verifier.py se usa en auditoría (launcher.py opción [3])**
   ```
   is_healthy = verifier.check_health(filepath)
   if is_healthy:
       print("✅ Sano")
   else:
       write_to_broken_list(filepath)  # Reportar en informe
   ```
   Esta función NO borra nada. Solo reporta.

---

🎯 **Principio de Seguridad**

El verifier funciona con una **presunción de culpabilidad**:
- Archivo nuevo = CULPABLE hasta demostrar inocencia
- Solo 1 fallo en cualquier prueba = VEREDICTO NEGATIVO
- Si hay duda = NO borra el original

**Resultado:** En 15.000+ archivos procesados, 0 pérdidas de datos.

---

📝 **Resumen**

El verifier.py es un filtro de seguridad multinivel. Es como tener un perito forense que:
1. Analiza la estructura del archivo
2. Verifica que sea decodificable
3. Compara original vs nuevo
4. Detecta spam en metadatos
5. Generate un veredicto JSON que otros scripts respetan

Su filosofía: **Seguridad total. Ante la duda, preservar el original.**

📋 Plan de Verificación: "El Auditor Universal" (Offline & Strict)

El objetivo es crear un módulo que pueda ser llamado por extract.py, rescuer.py o manualmente (launcher.py), devolviendo un veredicto fiable sin depender de internet ni de bases de datos externas.
1. Detección de Duplicados (Modo Forense Local)

    Enfoque: Búsqueda exclusiva dentro de la estructura de carpetas local. Nada de preguntar a Radarr.

    Lógica:

        Se analiza carpeta por carpeta (ámbito serie/temporada).

        Firma Rápida: Si Tamaño (bytes) + Duración (ms) coinciden -> Sospechoso.

        Juez MD5: Si hay sospecha, se calcula MD5 completo.

        Acción: Solo reportar. El usuario decide qué borrar. Esto detecta esos momentos donde Sonarr importó dos veces el mismo archivo con distinto nombre.

2. Higiene de Metadatos (Detector de Spam)

    Enfoque: Limpieza y coherencia interna. Cero APIs.

    Herramienta: mkvpropedit o MediaInfo (lectura rápida de headers).

    Lógica:

        Lista Negra: Buscar palabras clave en el título interno ("rarbg", "www.", "torrent", "core", "ac3-evo").

        Coherencia Simple: Si el archivo se llama Matrix.mkv y el título interno es Shrek, se marca como "Mismatch Crítico".

        Objetivo: Identificar archivos "sucios" para que un futuro proceso (o tú a mano) limpie los tags con un comando.

3. Gestión de Episodios Agrupados (Multi-Part)

    Enfoque: Respeto al archivo original (No cortar, solo validar).

    Lógica:

        Detección: Usar Regex en el nombre del archivo (S01E01-E02, 1x01-02).

        Ajuste de Tolerancia: Si se detecta patrón multi-episodio, el Verificador desactiva la alarma de "Duración excesiva" o ajusta la expectativa (ej: espera ~40-50 min en vez de ~20 min para un anime).

        Validación: Confirmar simplemente que el archivo es reproducible y no está cortado a la mitad.

4. ISO vs MKV (La Prueba del Algodón)

    Enfoque: Paridad Total de Contenido.

    Herramientas: makemkvcon info (origen) vs MediaInfo (destino).

    Lógica Estricta (Tu requisito 7.1.5):

        Contar pistas de Audio en ISO vs MKV. (Ej: 3 en ISO -> deben ser 3 en MKV).

        Contar pistas de Subtítulos en ISO vs MKV.

        Veredicto: Si falta una sola pista, el proceso de extracción se marca como FALLIDO. No borramos la ISO. El usuario debe decidir si le vale o si hubo error.

5. Comparativa de Salud (Legacy vs. Rescue / Segunda Opinión)

    Enfoque: Asegurar que la "cura" no ha matado al paciente.

    Lógica:

        Comparar Duración (tolerancia 5%).

        Comparar Resolución (Ancho no debe bajar).

        El "Doctor House" (MediaInfo): Si ffprobe dice que el archivo está roto o no da datos, consultamos a MediaInfo. Si MediaInfo lo lee bien, salvamos el archivo. Si ambos fallan, se declara Corrupto.

6. Integración Universal (Output Estructurado)

    Enfoque: El Verifier es un servidor de verdad, no un script de texto.

    Cambio Clave: En lugar de solo imprimir en pantalla, devolverá un Diccionario de Datos (JSON) a quien lo llame (extract.py o rescuer.py).

    Estructura del Output:
    JSON

{
  "valid": false,
  "errors": ["TRACK_COUNT_MISMATCH", "TITLE_SPAM"],
  "details": {
     "iso_audio_tracks": 4,
     "mkv_audio_tracks": 3
  }
}

Esto permite que el script extract.py lea valid: false y aborte el borrado del archivo avi, mp4, wmv o lo que fuese.
